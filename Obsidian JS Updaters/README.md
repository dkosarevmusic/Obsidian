# Утилита для массового обновления заметок в Obsidian

Этот набор скриптов предназначен для автоматизации рутинных задач по обновлению и исправлению большого количества файлов в вашем хранилище Obsidian.

## 📝 Назначение

Основная задача утилиты — помочь поддерживать консистентность и актуальность ваших заметок, особенно тех, которые используют скрипты `dataviewjs` или определенные метаданные (YAML frontmatter).

## 🚀 Первоначальная настройка

Перед первым запуском необходимо выполнить один простой шаг:

1.  **Указать путь к хранилищу:**
    *   Откройте файл `vault_config.yml`.
    *   Внутри, в поле `vault_path`, укажите **полный абсолютный путь** к вашему хранилищу Obsidian.

    Пример:
    ```yaml
    # vault_config.yml
    vault_path: 'C:/Users/ИмяПользователя/Documents/МоеХранилище'
    ```

2.  **Установить зависимости:**
    *   Для работы скрипта требуется библиотека `PyYAML`. Если она не установлена, откройте терминал или командную строку и выполните:
    ```bash
    pip install PyYAML
    ```

## 🛠️ Как это работает

Вы запускаете главный скрипт `obsidian_updater.py`, который предлагает вам выбрать одну из нескольких операций. Для каждой операции скрипт сначала анализирует файлы в вашем хранилище, а затем предлагает либо просто создать отчет, либо выполнить изменения.

### Безопасность

**Важно:** Перед тем как изменить любой файл, скрипт автоматически создает его полную копию (бэкап). Все бэкапы сохраняются в папку `Archive`, которая создается рядом с вашим хранилищем. Внутри нее будет папка с датой и временем запуска, где и будут лежать оригиналы измененных файлов. Вы всегда сможете восстановить их оттуда.

## ⚙️ Основные операции

После запуска `python obsidian_updater.py` вам будет предложено выбрать одно из следующих действий:

### 1. 🔄 Заменить код-блоки dataviewjs

Эта операция находит `dataviewjs` скрипты в ваших заметках и заменяет их на эталонный код из файла-шаблона. Это невероятно полезно, когда вам нужно обновить логику в десятках или сотнях заметок одновременно.

**Как настроить:**
*   Создайте свой `.yml` файл конфигурации (например, `my_project.yml`) по аналогии с `project.yml`.
*   В нем укажите:
    *   `reference_file_path`: Путь к файлу `.md`, из которого нужно скопировать блок `dataviewjs`.
    *   `target_types`: Список "типов" заметок, которые нужно обновить. Скрипт будет искать файлы, у которых в метаданных (YAML frontmatter) указан `type`, совпадающий с одним из значений в этом списке.
    *   `report_file_name`: Имя файла для отчета.

### 2. 🧹 Удалить разделители '---' после код-блоков

Иногда после `dataviewjs` блока может оставаться лишний разделитель `---`, который портит внешний вид заметки. Эта операция автоматически находит и удаляет такие разделители. Она определяет целевые файлы на основе `target_types` из всех других `.yml` конфигов.

### 3. 🛠️ Исправить поле 'status' (из списка в строку)

Некоторые плагины или старые версии скриптов могли сохранять статус в виде списка (`status: [в работе]`). Эта операция находит такие случаи и преобразует их в обычную строку (`status: в работе`), беря первый элемент из списка.

### 4. 🔍 Проверить тип поля 'status' (должен быть строкой)

Эта операция сканирует все файлы в хранилище и создает отчет, перечисляя все заметки, в которых поле `status` не является строкой (например, является списком, числом или другим типом). Сами файлы при этом не изменяются.

## ▶️ Процесс работы

1.  Откройте терминал или командную строку в папке со скриптами (`Obsidian JS Updaters`).
2.  Запустите главный скрипт: `python obsidian_updater.py`.
3.  Выберите номер операции из предложенного меню.
4.  Если это операция замены, выберите нужный `.yml` файл конфигурации.
5.  Скрипт проанализирует файлы и покажет, сколько файлов будет затронуто.
6.  Выберите режим:
    *   `1`: Только сгенерировать отчет в формате `.md` без внесения изменений.
    *   `2`: Выполнить операцию (с созданием бэкапов) и сгенерировать отчет.
7.  Если вы выбрали режим `2`, скрипт попросит вас подтвердить действие, введя `yes`. Это последняя точка невозврата.
8.  После завершения операции прочтите отчет, чтобы увидеть, какие файлы были изменены.
